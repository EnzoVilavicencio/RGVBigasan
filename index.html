💡 How Restocking Works
When you restock a product, enter how many units you bought and the total amount you paid for that batch. The system will automatically: (1) add units to current stock, (2) update the unit cost to reflect your latest purchase price, and (3) record the expense so it shows up in your profit calculation.

🚀 Deploy Guide

⚠️ Data is stored in your browser's local storage by default. For shared access across multiple devices, deploy to GitHub Pages or Netlify — but note that local storage is per-device. See the multi-device tip below.

Option A: GitHub Pages (Free, Permanent)
1
Go to github.com, create a free account.
2
Create a new repository named tindahan-pos, set to Public.
3
Save this page as index.html (File → Save Page As).
4
Upload index.html to the repo → Settings → Pages → Source: main branch → Save.
5
Your POS is live at https://YOUR-USERNAME.github.io/tindahan-pos 🎉

Option B: Netlify Drop (30 seconds)
1
Save page as index.html.
2
Go to netlify.com/drop, drag and drop the file. Done!

📱 Multi-Device Shared Data Tip
Since this is a static HTML file, each device has its own local storage. For a single-device setup (one tablet/laptop at the counter), everything works perfectly.

For true multi-device sync (e.g. phone + laptop both updating data), you would need a backend database — which requires a paid service (Firebase, Supabase). For most sari-sari stores, one shared device is the recommended setup.

Workaround: Use Export CSV to transfer data between devices when needed.

• Export CSV regularly to back up data.
• Best on a tablet or laptop at the counter.
• Clearing browser data deletes all records — back up often!
