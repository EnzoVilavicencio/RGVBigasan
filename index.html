💡 How Restocking Works
When you run out of stock and buy more from your supplier, log it here. Each restock entry:
• Adds units to the product's current stock automatically
• Updates the unit cost of the product to the new purchase price
• Records the total expense (units × new unit cost) which is deducted from your monthly profit on the Dashboard under "Restock Expenses" — separate from the Cost of Goods sold

🚀 Deploy & Share Online (Free)

ℹ️ This app uses your browser's localStorage. To share data across multiple devices (multiple users on different phones/tablets), you need to host it AND use the same browser on the same device — OR use the Supabase upgrade path below for true multi-device sync.

Option A: GitHub Pages (Free Static Hosting)
1
Go to github.com, create a free account.
2
Create a repo named tindahan-pos (Public).
3
Save this page as index.html (File → Save Page As).
4
Upload index.html to the repo. Go to Settings → Pages, set source to main branch. Done!

Option B: Netlify Drop (30 seconds, no account)
1
Save as index.html. Go to netlify.com/drop. Drag and drop. Get instant URL.

⭐ Option C: True Multi-Device Sync (Supabase — Still Free)
For true multi-user access from different devices, ask your developer to integrate Supabase (free tier) as the database backend. This replaces localStorage with a real cloud database. Supabase free tier supports up to 500MB data — more than enough for a sari-sari store.
• Export CSV regularly to back up your data.
• Best used on a tablet or laptop at the counter.
• Clearing browser cache will erase localStorage data — export often!
